# RB-ITSM-001 — ITSM Ticketing and Shift Handoff Standard

## Domain
ServiceNow, Remedy, incident tickets, SLA, assignment group, shift turnover, next action.

## Trigger pattern
Use this runbook for any unresolved, reviewed, blocked, escalated, or physical-dispatch incident.

## Ticketing requirements
Every ticket must include:
- Source incident ID
- ITSM platform
- Priority
- Assignment group
- SLA
- Owner or next-shift action
- Context from correlation, classification, and governance

## Governance rule
Human review items must be ticketed before shift handoff. Blocked automation must show the reason and required approval path.

## Handoff guidance
The next-shift brief must include root cause, what was auto-resolved, what was escalated, what was blocked, open tickets, owner, SLA, and next action.

## Learning update
The post-incident learner should propose handoff template improvements when ownership, SLA, or next action is missing.
